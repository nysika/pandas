💻 Техническая задача для программиста

Используя pandas (и при необходимости numpy, datetime, itertools), необходимо:

  • Загрузить sales_data.csv.
  • Рассчитать для каждой строки:
    • revenue = unit_price * quantity_sold * (1 - discount)
    • cost_price (вычисляется как unit_price * 0.6 для всех товаров)
    • profit = revenue - cost_price * quantity_sold
  • Сгруппировать данные по product_id и создать признаки:
    📦 total_quantity 💰 total_revenue 🏷️ avg_discount 🌍 unique_regions 📅 sales_frequency
  • Выполнить региональный анализ:
    • Для каждого region найти топ-2 товара по сумме profit и сохранить в отдельном DataFrame.
    • Определить регионы, где profit от сегмента Премиум составляет более 40% от общего profit региона.
  • Построить прогноз (упрощённый, но правдоподобный):
    • Для каждого товара экстраполировать продажи на следующие 7 дней, используя линейную регрессию (через sklearn или numpy.polyfit) по дням исходных продаж.
    • Если для товара меньше 3 уникальных дней продаж — использовать медианное значение quantity_sold за все дни.
  • Применить кластеризацию (k-means, через sklearn) для товаров на основе признаков:
    📊 total_quantity 💵 total_revenue 🎯 avg_discount 🗺️ unique_regions 🔮 прогноз на 7 дней
    Количество кластеров определить методом локтя (Elbow method), результат — новый столбец cluster_id в итоговой таблице товаров.
  • Создать итоговый отчёт — Excel-файл со следующими листами:
    📋 products_summary
    агрегация по товарам + кластер + прогноз
    🗺️ region_profit_analysis
    прибыль по регионам и сегментам
    ⭐ high_value_regions
    регионы, где доля Премиум > 40%
    💡 cluster_recommendations
    средние показатели и текстовая стратегия
⚠️ Обработать крайние случаи:
  • Если в файле есть товары с нулевыми продажами — исключить из кластеризации, но включить в отчёт с пометкой inactive.
  • Добавить проверку на отрицательную прибыль (если discount > 1).
